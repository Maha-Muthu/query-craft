Introduction to Databases


What are databases ?

Databases are organized collections of information. While they can be physical collections like a hospital's patient files, in the context of websites and applications, they refer to digital systems managed by Database Management Systems (DBMS). These systems allow users to store, retrieve, and manage data efficiently.


Relational Databases

Relational databases use tables to structure data into rows and columns. Each table includes a primary key to uniquely identify each row. This model supports complex queries through SQL, making it ideal for structured data like employee records or sales data.


Non-Relational Databases

Non-relational databases, or NoSQL databases, are designed for unstructured data and can scale horizontally. They include various models such as key-value stores, document stores, and columnar databases, each suited for different types of data and applications.


Applications of Databases

Databases are vital for managing customer information, inventory, and financial data in businesses, ensuring smooth operations. In healthcare, they store patient records and treatment histories, improving care quality. Academic institutions use databases to manage student records and course information. E-commerce platforms rely on them to track products, customer orders, and transactions. Telecommunications companies use databases for billing systems and customer service records. In finance, databases help manage accounts, transactions, and regulatory compliance.


Database Management and Security

Managing databases involves tools like command line clients and graphical interfaces. As databases grow, techniques like replication and sharding are used to handle large data volumes. Security measures, such as TLS encryption, are essential to protect data in transit and at rest.

Query Craft

Overview

Query Craft is an educational tool designed to guide students on effectively crafting SQL queries. It provides interactive learning materials, examples, and practice problems to help users understand various SQL concepts and improve their query writing skills.

Components

Query Craft focuses on two significant aspects: Learning Lab Material and Query Practice.

The Lab Material contains learning material along with examples. It covers basic data manipulation, aggregation, joins, triggers, and advanced queries.

The Query Practice includes five schemas with tables and data loaded and contains a set of queries to practice; the solution, along with the expected results and explanations, is also provided to improve learning.

How Query Craft Runs: Dockers

What is a Docker?

Docker is like a magic box for your software! It's a tool that helps you pack up an entire application and everything it needs to run smoothly, like libraries and settings, into a neat, portable package called a container. Imagine it's like a mini-virtual computer just for your app!
Docker is a platform that enables developers to package, deploy, and run applications within containers. Containers are lightweight, portable, and self-sufficient environments that contain everything needed to run an application, including code, runtime, libraries, and dependencies. Docker uses containerization technology to ensure consistency across different computing environments.

What are Docker Volumes ?

Docker Volumes are like special storage bins for your Docker containers! They let you save data outside your containers so that even if you stop or delete a container, the data is safe and sound. Imagine a Docker container is like a kitchen, and Docker Volumes are like a refrigerator. You can cook different meals (run different applications) in the kitchen, but the food and ingredients (data) are kept safe in the refrigerator. Even if you change the kitchen setup or clean it up (stop or remove a container), the food in the refrigerator remains untouched. When you set up a new kitchen or start cooking again (start a new container), you can access all the ingredients you saved in the refrigerator.
Docker Volumes are a mechanism for persisting data generated and used by Docker containers. Unlike the container's writable layer, data in volumes isn't tied to the lifecycle of a container, making it more durable and allowing it to be shared between multiple containers. Volumes are managed by Docker and can be stored on the host filesystem or a remote machine, providing flexibility and consistency for data storage in containerized applications.

What is Docker Compose ?

Docker Compose is like a conductor for an orchestra of Docker containers! It helps you organize and manage multiple containers that need to work together to run a full application. With Docker Compose, you can write a simple file that lists all the containers your app needs and how they should interact, and then start them all with a single command. It's like having a remote control to turn on everything at once!
Docker Compose is a tool that allows you to define and run multi-container Docker applications. Using a YAML file, developers can specify the services, networks, and volumes required for an application and start everything with a single command. This makes it easier to manage complex applications with multiple interconnected containers, ensuring they run consistently across different environments.

Query Draft setup is containerized using Docker, which packages the Flask application and the PostgreSQL database into separate, connected containers. Docker Compose handles the setup, ensuring efficient communication between the Flask server and the PostgreSQL database. Docker volumes store the schema for the practice material.
