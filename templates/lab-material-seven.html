Java Web Development: Servlet, JSP, and Tomcat Integration

This material provides a comprehensive guide on web database programming using Java, JSP (Java Server Pages), and Tomcat. Key principles include the separation of JSP for the interface and Servlet for application flow control and logic processing. Practical exercises involve creating a web application project in IntelliJ and deploying a simple "Hello World" web application using Tomcat.

Principle for programming Servlet + JSP

  • Never allow users to directly access to your JSP page.
  • JSP is only considered as the place to display interface.
  • Servlet acts as the controller of the application flows and program logical processing.
  • Open the JDBC connection and transaction management in Filter (Optional).

Integrate Tomcat with IntelliJ

  • Once you are sure that Tomcat runs as expected, shut it down.
  • Start it from within IntelliJ.
  • Go to IntelliJ and in Preferences → Plugins.
  • Search and install the plugin Smart Tomcat.

Create Web Application Project

  • Create a web project in IntelliJ
  • Select: File → New → Project
  • Select: Maven Archetype → maven.archetype.webapp (in the archetype dropdown menu)
  • You can test your 'Hello World' web application can run with the local Tomcat - you can use the 'run' button
