Periodic Table Schema

The PeriodicTable schema is a structured representation of chemical elements, capturing key properties such as atomic number, name, and symbol, along with atomic mass and subatomic particle counts. It includes details on the element's period, phase (like gas or solid), natural occurrence, and type (e.g., nonmetal or noble gas). Each element is uniquely identified by its atomic number, ensuring precise and organized data management.

1. List the names of nonmetals('Nonmetal') that are gases('gas').

Expected Results:

element_name
Hydrogen
Oxygen
Nitrogen
Fluorine
Chlorine

Query: SELECT element_name FROM periodic_table.periodic_table WHERE element_Type = 'Nonmetal' AND Phase = 'gas';

Explanation: The query filters the periodic table data by checking for elements where element_type is 'Nonmetal' and phase is 'gas'. By using the WHERE clause, it ensures that only rows matching both conditions are considered. The SELECT statement then extracts the element_name for these specific rows.

2. What are the top 5 elements with the highest number of neutrons?

Expected Results:

element_name number_of_neutrons
Tennessine 178
Oganesson 176
Livermorium 176
Flerovium 175

Query: SELECT element_name, number_of_neutrons FROM periodic_table.periodic_table ORDER BY number_of_neutrons DESC LIMIT 5;

Explanation: Retrieves the top 5 elements with the highest number of neutrons by first ordering the elements in descending order using ORDER BY number_of_neutrons DESC. The LIMIT 5 clause then restricts the result to the first five rows, which correspond to the elements with the most neutrons.

3. Find the lightest element in each phase.

element_name lightest_element_mass phase
Hydrogen 1.007 gas
Lithium 6.941 solid
Bromine 79.904 liq
Technetium 98 artificial

Query: SELECT phase, element_name, atomic_mass AS Lightest_Element_Mass FROM periodic_table.periodic_table WHERE (phase, atomic_mass) IN ( SELECT phase, MIN(atomic_mass) FROM periodic_table.periodic_table GROUP BY phase );

Explanation: Identifies the lightest element in each phase by first determining the minimum atomic mass within each phase using GROUP BY phase and MIN(atomic_mass) in a subquery. The main query then matches these minimum atomic masses with their corresponding phases to retrieve the element_name, phase, and atomic mass (aliased as Lightest_Element_Mass).

6. Find Elements with More Protons Than Neutrons.

Expected Results:

element_name number_of_neutrons number_of_protons

Query: SELECT element_name, number_of_neutrons, number_of_protons FROM periodic_table.periodic_table WHERE number_of_protons > number_of_neutrons;

Explanation: The query identifies elements with more protons than neutrons by using the WHERE clause to filter rows where the number_of_protons is greater than the number_of_neutrons. The query then selects and returns the element_name, along with the counts of number_of_neutrons and number_of_protons, for these elements.

8. Find the Heaviest Element in Each Period.

Query: SELECT period_number, element_name, atomic_mass AS Heaviest_Element_Mass FROM periodic_table.periodic_table WHERE (period_number, atomic_mass) IN ( SELECT period_number, MAX(atomic_mass) FROM periodic_table.periodic_table GROUP BY period_number );

Explanation: The query finds the heaviest element in each period by first determining the maximum atomic mass for each period using GROUP BY period_number and MAX(atomic_mass) in a subquery. The main query then uses the WHERE clause to filter elements matching both the period number and the maximum atomic mass. It retrieves the period_number, element_name, and atomic_mass (aliased as HeaviestElementMass), ensuring that only the heaviest element for each period is returned.

9. Count the Number of Elements in Each Phase.

Expected Results:

number_of_elements phase
2 liq
11 gas
77 solid
28 artificial

Query: SELECT phase, COUNT(*) AS number_of_elements FROM periodic_table.periodic_table GROUP BY phase;

Explanation: The query counts the number of elements in each phase by grouping the elements based on the phase column using GROUP BY phase. The COUNT(*) function then counts the number of elements within each group, and the results are returned with the phase and the corresponding count, labeled as num_elements.
